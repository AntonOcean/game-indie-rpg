# Run 09 — Смерть врага и лут

**Фаза плана:** implementation-plan § «Фаза 6».

## Цель

При **`Health <= 0`**: компонент **`Dead`**, исключение из коллизий/атак, **скрытие/удаление** визуала врага через **RenderSystem** / реестр. Создать сущность **лута** (`Loot`, `Position`, `Hitbox`, `RenderRef`) — другой цвет квадрата. **Подбор:** пересечение AABB игрока и лута → удалить лут, инкремент счётчика или минимальный UI «+1».

## Входные условия

Run-08: враг, урон, кулдаун.

## Задачи (чек-лист)

- [ ] Система **death** в фиксированном порядке pipeline (до или после других систем — задокументировать).
- [ ] Мёртвые враги не принимают урон и не блокируют проход (или не участвуют в hit-test атаки — согласовать).
- [ ] Лут спавнится в позиции врага.
- [ ] Подбор при пересечении хитбоксов; состояние счётчика/ UI обновляется.
- [ ] Очистка узлов реестра для удалённых сущностей без утечек.

## Ограничения

- Один тип лута (MVP); не инвентарь и не несколько типов.
- Не обязательно решать UX «случайный подбор мимо» — по плану это после MVP; при необходимости краткий комментарий в коде.

## Как проверить

Полный цикл вручную: подойти → бить до нуля → лут появляется → подобрать → счётчик/UI меняется.

## Выход для следующего рана

Замкнутый геймплейный цикл MVP; остаётся полировка run-10.
