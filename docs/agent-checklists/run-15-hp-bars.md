# Run 15 — Полоски здоровья (HP bars)

**Фаза плана:** post-mvp-development-plan § «Фаза 1.2».

## Цель

Визуальная обратная связь по здоровью: **HP bar над врагами** (screen-space, без subpixel jitter) и **HP игрока в HUD** (фиксированный угол экрана). Данные из ECS-компонента `Health`.

## Входные условия

Run-14: анимации работают, RenderAdapter введён.

## Ключевые ссылки из architecture.md / post-mvp

- **Screen-space слой** предпочтителен: `worldToScreen` → `Math.round` → рендер.
- Мир + float позиция + lerp камеры + дочерний контейнер = **subpixel jitter**.
- HP bar — **presentation**, не мутирует ECS.

## Задачи (чек-лист)

- [x] **Screen-space контейнер** для HP баров (`src/render/hpBarLayer.ts` или аналог):
  - Отдельный `Container`, добавленный **поверх** worldRoot в stage.
  - Не является дочерним узлом worldRoot (не масштабируется камерой).
- [x] **HP bar врага:**
  - Для каждой живой сущности `Enemy` с `Health`: полоска над спрайтом.
  - Позиция: `worldToScreen(Position.x, Position.y)` с учётом камеры и worldScale → `Math.round(x)`, `Math.round(y)` — **snap к пикселю**.
  - Ширина фона (серый) = `BAR_WIDTH`; ширина заполнения (зелёный→жёлтый→красный) = `BAR_WIDTH * (currentHP / maxHP)`.
  - Цвет: ≥ 60% → зелёный; ≥ 30% → жёлтый; < 30% → красный.
  - Скрывать бар, если HP = maxHP (враг не повреждён) — опционально.
- [x] **HP bar / индикатор игрока:**
  - Фиксированная позиция в углу экрана (например левый верхний, с отступом).
  - Аналогичная полоска или текст `HP: 100/100`.
  - Сейчас у игрока может не быть `Health` компонента — **добавить**, если нет (maxHP из констант, например `PLAYER.MAX_HP = 100`).
- [x] **Обновление каждый кадр** (в presentation фазе):
  - Пересчёт позиции баров врагов по текущей позиции камеры.
  - Обновление ширины заполнения при изменении HP.
- [x] **Очистка:** при удалении сущности врага (смерть → лут) — удалить соответствующий HP bar из контейнера.
- [x] **Утилита `worldToScreen`:** если ещё нет — создать на основе камеры worldRoot (`worldRoot.toGlobal` или ручной расчёт по offset + scale).
- [x] **Константы** в `gameBalance.ts`:
  - `HP_BAR.WIDTH`, `HP_BAR.HEIGHT`, `HP_BAR.OFFSET_Y` (смещение над спрайтом).
  - `HP_BAR.COLOR_HIGH`, `HP_BAR.COLOR_MID`, `HP_BAR.COLOR_LOW`.

## Ограничения

- Не менять боевую логику.
- Не реализовывать урон по игроку (AI) — run-19; но компонент `Health` для игрока завести.
- HP bar — **read-only** относительно ECS.

## Как проверить

1. Над неповреждённым врагом бар скрыт (или полностью зелёный).
2. Удар по врагу → бар уменьшается, цвет меняется.
3. Бар **не дёргается** при движении камеры (pixel-snap).
4. HP игрока видно в углу (сейчас не меняется — меняться будет с run-19).
5. Смерть врага → бар исчезает без ошибок.
6. Resize окна → бары пересчитывают позицию корректно.
