# Run 07 — Камера: worldRoot, worldScale, clamp

**Фаза плана:** implementation-plan § «Фаза 4».

## Цель

Обернуть карту и игровые объекты в **`worldRoot`**; ввести **`worldScale`**; камера следует за игроком с **clamp** к границам карты в **мировых пикселях**; **`viewW` / `viewH`** считать как `canvasSize / worldScale`; обновлять при **resize** и изменении viewport TWA.

## Входные условия

Run-06: движущийся игрок, `worldRoot` согласован с формулой `screenToWorld`.

## Задачи (чек-лист)

- [ ] Явно: `mapWidthPx`, `mapHeightPx` из тайлов × размер тайла.
- [ ] Каждый кадр (или при движении игрока + resize): вычислить `viewW`, `viewH`, `desiredLeft` / `desiredTop`, `camX` / `camY` с `clamp` (implementation-plan).
- [ ] Обработать случай карты меньше viewport (letterbox / центрирование по политике из плана).
- [ ] Убедиться, что **ввод** (`screenToWorld`) использует **те же** `worldRoot.position` и `worldScale`, что и камера.
- [ ] Проверить смену ориентации и масштаба (если `worldScale` меняется из кода).

## Ограничения

- Не добавлять врагов и лут.

## Как проверить

1. Игрок остаётся в кадре при ходьбе к краям карты; камера **не уезжает** за границы мира.
2. Resize окна — картинка и ввод остаются согласованными (клик/тап в том же месте мира).
3. Нет расхождения «персонаж не под курсором» после движения камеры.

## Выход для следующего рана

Стабильная камера и единая система координат для боя.
