# Run 06 — Ввод → Intent, dt, движение, коллизии, screen→world

**Фаза плана:** implementation-plan § «Фаза 3» (ввод, время, коллизии, screen→world).

## Цель

Ввести **`PlayerIntent`**, сборщик ввода (**desktop**: целевая точка / стрелки → `moveTo`; **mobile**: направление с **явной остановкой** — удержание или короткий шаг, зафиксировать в коде). Применить **dt** из тикера Pixi (секунды, опционально кап на лаг). Реализовать **`isBlockedTile`**, **`hitboxIntersectsBlockedTiles`** (четыре угла), разрешение коллизий: пробное смещение по **X**, затем **Y**, откат оси. Одна функция **`screenToWorld`** для pointer и будущей атаки (implementation-plan, формула с `worldRoot` и `worldScale` — если `worldScale` ещё 1, заложить параметры).

## Входные условия

Run-05: игрок, карта, `collisionData`, размеры тайлов, `worldRoot` (даже без финальной камеры).

## Задачи (чек-лист)

- [ ] Тип `PlayerIntent` и агрегация на кадр: не смешивать сырые события с логикой ECS напрямую.
- [ ] Desktop: движение к точке (`moveTo`) и/или стрелки — согласно implementation-plan.
- [ ] Mobile: выбран один вариант остановки (удержание / короткий шаг) и описан комментарием; без бесконечного дрейфа.
- [ ] `dtSec` из `Ticker.deltaMS` / 1000; скорость в единицах/сек; опционально кап `dt`.
- [ ] `isBlockedTile(tx, ty)` с границами карты и проверкой `collisionData[ty * mapWidth + tx] !== 0` (или согласованное правило ненулевого gid).
- [ ] Проверка хитбокса: **все четыре угла** → тайлы; откат по оси при пересечении.
- [ ] `screenToWorld` единообразно с позицией камеры/масштабом (даже если камера статична, заложить правильные параметры).
- [ ] Система разрешения намерения → скорость/цель → обновление `Position` → коллизии.

## Ограничения

- Не добавлять врага и атаку — run-08.
- Не финализировать камеру follow — run-07 (но world→экран для ввода должен быть согласован с текущим `worldRoot`).

## Как проверить

1. **Desktop:** стрелки / клик — персонаж ходит по проходимым клеткам, **упирается в стены** слоя `collisions`.
2. **Mobile (эмуляция или устройство):** отпускание пальца останавливает direction-based движение (если выбран режим удержания).
3. Нет проскоков углов хитбокса сквозь углы стен (базовая проверка глазами).
4. Консоль без ошибок при движении и resize.
