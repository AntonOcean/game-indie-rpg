# Run 11 — Константы баланса и Debug Overlay

**Фаза плана:** post-mvp-development-plan § «Микро-улучшения».

## Цель

Собрать все «магические числа» из кодовой базы в один файл констант (`src/constants/gameBalance.ts`). Добавить debug overlay с переключением по клавише **D**: хитбоксы, HP, позиции сущностей. Это **фундамент** для всех следующих фаз — удобная отладка AI, анимаций, лута.

## Входные условия

MVP (run-01 — run-10): игрок, враг, бой, лут, камера.

## Задачи (чек-лист)

- [ ] Создать файл `apps/client/src/constants/gameBalance.ts`.
- [ ] Перенести в него все числовые значения из кода:
  - `PLAYER.SPEED`, `PLAYER.HITBOX_SIZE`, `PLAYER.ATTACK_RANGE`, `PLAYER.ATTACK_COOLDOWN_MS`, `PLAYER.ATTACK_DAMAGE`
  - `ENEMY.HP`, `ENEMY.HITBOX_SIZE`, `ENEMY.VISUAL_SIZE`, `ENEMY.VISUAL_COLOR`
  - `LOOT.HITBOX_SIZE`, `LOOT.VISUAL_SIZE`, `LOOT.VISUAL_COLOR`
  - `CAMERA.WORLD_SCALE`, `CAMERA.CLAMP_MARGIN`
  - `MAP.TILE_SIZE` (если хардкожен)
- [ ] Заменить литералы в исходных файлах на импорты из `gameBalance.ts`.
- [ ] Создать модуль debug overlay (`apps/client/src/debug/debugOverlay.ts`):
  - Контейнер поверх сцены (или screen-space layer).
  - Отрисовка AABB хитбоксов всех сущностей (прозрачные прямоугольники).
  - Текст HP над врагами.
  - Текст `Position` (x, y) рядом с игроком.
- [ ] Переключатель: клавиша **D** — показать / скрыть overlay.
- [ ] Overlay **не** влияет на ECS / симуляцию — только чтение.

## Ограничения

- Не менять игровую логику — только вынос констант и добавление визуализации.
- Не добавлять новые ECS-компоненты.

## Как проверить

1. `npm run dev:client` — игра работает **идентично** до и после рефакторинга.
2. Нажать **D** — хитбоксы видны, HP отображается, позиция игрока обновляется.
3. Повторное нажатие **D** — overlay исчезает.
4. Бой, движение, подбор лута без регрессий.

## Выход для следующего рана

Единый файл констант; debug overlay для визуальной отладки будущих систем (AI, анимации, лут).
