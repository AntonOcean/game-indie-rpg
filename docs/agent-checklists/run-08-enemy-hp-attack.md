# Run 08 — Враг, HP, атака по тапу/клику, кулдаун

**Фаза плана:** implementation-plan § «Фаза 5».

## Цель

Сущность врага: **`Enemy`**, **`Health`**, **`RenderRef`**, **`Hitbox`**; визуал через реестр (**красный квадрат**). Ввод: `pointertap` → **`screenToWorld`** → hit-test врага по AABB в мировых px → **`PlayerIntent.attackTarget`**; иначе — обновление полей движения (как в run-06). Система атаки: урон **только** если цель живая, дистанция **\< attackRange**, кулдаун истёк (~**300 ms** константа). При успехе: урон, старт кулдауна, вызов **no-op `ATTACK`** протокола.

## Входные условия

Run-07: камера, ввод, игрок.

## Задачи (чек-лист)

- [ ] Спавн одного (или нескольких) врагов на карте в проходимой зоне.
- [ ] Приоритет ввода: сначала проверка попадания по врагу, затем движение.
- [ ] Компонент или состояние **`AttackCooldown`** у игрока.
- [ ] Все условия атаки из implementation-plan перечислены в коде явно.
- [ ] Эмит события протокола (заглушка) при успешной атаке.

## Ограничения

- Не реализовывать смерть и лут — run-09.
- Не наносить урон без выполнения всех условий.

## Как проверить

1. В радиусе: несколько ударов с паузой уменьшают HP (визуально или через debug UI/log).
2. Издалека: тап по врагу **не** наносит урон.
3. Кулдаун соблюдается (спам не убивает мгновенно сверх ожидаемого DPS).
4. Враг остаётся на карте с корректной отрисовкой через RenderSystem.
