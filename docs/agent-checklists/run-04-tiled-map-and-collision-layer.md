# Run 04 — Карта Tiled: pixi-tiledmap и слой коллизий

**Фаза плана:** implementation-plan § «Фаза 2» (2.1–2.2).

## Цель

Загрузить `map.tmj` через цепочку с путями **`/assets/...`**, отрисовать слои **`ground`**, **`wall`**; слой **`collisions`** не показывать; сохранить ссылки на **`collisions.data`**, размер карты в тайлах, `tilewidth` / `tileheight` для будущих систем.

## Входные условия

Run-01–03; в `public/assets/` лежат `map.tmj` и все внешние тайлсеты с **абсолютными от корня сайта** путями (implementation-plan 2.1).

## Задачи (чек-лист)

- [ ] Зарегистрировать расширения **pixi-tiledmap** для `Assets` Pixi v8 (по README пакета).
- [ ] Загрузка: `fetch('/assets/map.tmj')` или `Assets.load` с тем же URL.
- [ ] Собрать визуальные слои в порядке: ground → wall; **collisions** скрыть (`visible = false` или не монтировать).
- [ ] Экспортировать в игровой код (модуль/контекст): `collisionData` (линейный массив), `mapWidth`, `mapHeight` (в тайлах), `tileWidth`, `tileHeight`.
- [ ] Проверить в DevTools **Network**: все запросы к карте и тайлсетам — **200**.

## Ограничения

- Не подключать bitECS и движение в этом ране.
- Не дублировать логику `isBlockedTile` здесь, если не нужно для отладки — достаточно сохранить данные; утилита коллизий — run-06.

## Как проверить

1. Запуск dev: карта **видна** (целиком или частично — камера будет в run-07).
2. Слой коллизий **не** виден игроку.
3. В консоли можно временно залогировать длину `collisionData` и размеры карты — согласованы с ожиданиями из Tiled.

## Выход для следующего рана

Карта на сцене Pixi + стабильные метаданные для ECS и коллизий.
