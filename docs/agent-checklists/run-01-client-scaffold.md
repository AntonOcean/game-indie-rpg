# Run 01 — Клиент: скaffold, зависимости, пустая сцена Pixi

**Фаза плана:** implementation-plan § «Фаза 0» (клиентская часть).

## Цель

Инициализировать клиент (Vite + TypeScript), подключить зависимости рендера и ECS, настроить раздачу ассетов через `public/`, показать **пустую** сцену Pixi на весь экран без ошибок в консоли. Скрипт Telegram Web App можно подключить **заглушкой** (тег в HTML без полной интеграции — полная интеграция в run-03).

## Входные условия

Репозиторий может быть пустым или без клиента — этот ран создаёт основу.

## Задачи (чек-лист)

- [ ] Инициализировать **Vite + TypeScript** (каталог по договорённости репо: например `apps/client` или `client` в корне — зафиксировать одну структуру).
- [ ] Установить: `pixi.js@8`, `pixi-tiledmap`, `bitecs` (как в implementation-plan).
- [ ] Обеспечить зеркало ассетов: **`public/assets/`** — сюда кладётся `map.tmj` и тайлсеты по структуре из репо (`assets/` → копия/симлинк в `public/assets/`, если исходники лежат отдельно).
- [ ] В `index.html` подключить `https://telegram.org/js/telegram-web-app.js` (достаточно тега; вызовы API — в run-03).
- [ ] В точке входа: создать `Application` Pixi v8, canvas на весь контейнер, пустой stage (или один тестовый `Graphics`, если нужен визуальный якорь — не обязательно).
- [ ] Зафиксировать npm-скрипты: как минимум `dev` / `build` для клиента (имена согласовать с run-02).

## Ограничения

- Не реализовывать загрузку карты, ECS и сервер — только клиентский скелет.
- Не менять семантику путей в `.tmj` здесь, если карты ещё нет — достаточно пустой `public/assets/` или копии из репо с путями `/assets/...` (правка путей — при появлении карты в run-04).

## Как проверить

1. `npm install` в корне или в клиентском пакете (как заведено).
2. `npm run dev` (или эквивалент) — страница открывается, **консоль без ошибок**.
3. Виден canvas / область игры на весь экран (или основной контейнер).

## Выход для следующего рана

Рабочий клиентский билд, зависимости в lockfile, `public/assets/` готов принимать карту.
