# Run 12 — Спрайты персонажей (замена квадратов)

**Фаза плана:** post-mvp-development-plan § «Фаза 1.1» (первая часть — ассеты и статичное отображение).

## Цель

Заменить цветные квадраты игрока и врага на спрайты из пака `full-assets/Tiny RPG Character Asset Pack v1.03 -Free Soldier&Orc`. На этом этапе — **только статичный кадр** (idle, первый фрейм); анимация — следующие раны.

## Входные условия

Run-11: константы вынесены, debug overlay работает.

## Контекст по ассетам

Структура пака (на диске, путь с `&`):
- `full-assets/Tiny RPG Character Asset Pack v1.03 -Free Soldier&Orc/Characters(100x100)/Soldier/Soldier/` — листы: `*-Idle.png`, `*-Walk.png`, `*-Attack01.png`, `*-Hurt.png`, `*-Death.png`.
- Аналогично `…/Orc/Orc/` — листы орка.
- Кадр: **100×100 px** на сетке листа.

## Задачи (чек-лист)

- [ ] Скопировать из `full-assets/…/Soldier/Soldier/` нужные PNG в `assets/characters/soldier/`:
  - `Soldier-Idle.png` (если не скопирован ранее).
  - Убедиться, что файл доступен по `/assets/characters/soldier/Soldier-Idle.png`.
- [ ] Аналогично `assets/characters/orc/` → `Orc-Idle.png`.
- [ ] Запустить `make assets-sync` (или ручное копирование в `apps/client/public/assets/`).
- [ ] Загрузка спрайт-листов через Pixi `Assets`:
  - Определить metadata для spritesheet (JSON или программно): размер кадра 100×100, количество кадров в idle-листе.
  - Создать `Spritesheet` или загрузить через `Texture` + `Rectangle` для первого кадра.
- [ ] В `playerVisual.ts` (или `mountPlayerVisual.ts`): заменить `Graphics` (синий квадрат) на `Sprite` из первого кадра idle солдата.
- [ ] В `enemyVisual.ts` (или `mountEnemyVisual.ts`): заменить `Graphics` (красный квадрат) на `Sprite` из первого кадра idle орка.
- [ ] Скорректировать anchor спрайтов (центр / низ — в зависимости от оси позиции).
- [ ] Скорректировать размер спрайта и хитбокс, если 100×100 слишком велик для тайла 32×32 при текущем `worldScale`. Подобрать `scale` спрайта.
- [ ] Debug overlay: хитбоксы по-прежнему отрисовываются корректно поверх спрайтов.

## Ограничения

- **Не** реализовывать анимацию (смена кадров) — run-13.
- **Не** менять логику боя, движения, подбора лута.
- Лут остаётся жёлтым квадратом.

## Как проверить

1. Игрок отображается как солдат (статичный кадр).
2. Враг отображается как орк (статичный кадр).
3. Хитбоксы (debug D) соответствуют видимой области спрайтов.
4. Бой, движение, подбор лута — без регрессий.
5. Спрайты не «плывут» относительно позиции (anchor корректен).

## Выход для следующего рана

Визуальная база для анимационной системы: спрайты загружены, отображены, размеры согласованы.
